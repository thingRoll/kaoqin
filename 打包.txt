### 📦 考勤算力系统 - 终极瘦身与极速启动打包指南

**【核心目标】**

1. 解决 `pandas` 附带海量无关库导致的 exe 体积过大（从 400MB 压缩至 70MB 左右）。
2. 解决单文件（`-F`）模式每次启动需解压导致的“启动慢、转圈圈”问题（实现点击图标秒开）。
3. 解决无控制台（`-w`）模式下的 `sys.stdout` 崩溃报错。

---

#### 第一步：代码防呆检查（仅需确认一次）

在打包前，确保 `attendance.py` 代码开头的输出重定向已加上 `None` 值保护（防止无控制台模式下崩溃）：

```python
# 强制输出编码 (增加防呆：防止在无控制台打包模式下sys.stdout为None导致报错)
if sys.stdout is not None:
    sys.stdout.reconfigure(encoding='utf-8')
warnings.filterwarnings('ignore')

```

#### 第二步：构建纯净的“无菌”打包环境

*目的：隔离电脑本地杂乱的 Python 库，防止 PyInstaller 乱抓壮丁。*

1. 打开代码所在的文件夹。
2. 在顶部文件路径地址栏中输入 `cmd`，然后按回车，打开黑色的命令提示符窗口。
3. 输入以下命令创建一个名为 `myenv` 的独立虚拟环境：
```bash
python -m venv myenv

```


4. 激活这个虚拟环境（成功后，命令行路径最前面会出现 `(myenv)` 字样）：
```bash
myenv\Scripts\activate

```



#### 第三步：安装最小化核心依赖

*目的：只安装我们代码里真刀真枪用到的那几个库。*

在带有 `(myenv)` 的命令行中，运行以下命令：

```bash
pip install pandas openpyxl pyinstaller

```

*(注：tkinter、threading、json、re、time 等均为 Python 官方自带标准库，无需额外安装)*

#### 第四步：执行“排他瘦身+极速启动”打包指令

*目的：放弃 `-F` 带来的启动延迟，并强制踢出 `pandas` 夹带的科学计算巨无霸。*

继续在刚才的命令行中，复制并执行以下终极指令：

```bash
pyinstaller -w --exclude-module matplotlib --exclude-module scipy attendance.py

```

**【指令参数解析】**

* `-w`：Window 模式，隐藏背后黑色的命令行控制台。
* `--exclude-module xxx`：强制排除模块。明确告诉打包器不要把 `matplotlib`（画图库）和 `scipy`（高级科学计算库）打包进来，能瞬间减掉几百兆体积。
* **为何不加 `-F**`：不加 `-F` 会生成文件夹模式。运行时不需要再向 C盘 Temp 目录解压，**启动速度提升 10 倍以上（几乎秒开）**。

#### 第五步：获取与发布最终程序

1. 打包完成后，在代码同级目录下会多出一个 `dist` 文件夹。
2. 进入 `dist`，您会看到一个名为 `attendance` 的**文件夹**（而不是孤零零的一个 exe）。
3. **分发给别人时**：请将整个 `attendance` 文件夹打包成 `.zip` 压缩包发送。
4. **别人如何使用**：告诉同事将压缩包解压后，找到里面的 `attendance.exe`，**右键 -> 发送到 -> 桌面快捷方式**。以后直接双击桌面快捷方式即可秒开使用。

---

**💡 日常维护提示**：
以后如果修改了 `attendance.py` 的代码需要重新打包，只需要重复 **第二步的激活环境 (`myenv\Scripts\activate`)** 和 **第四步的打包指令** 即可，第一步和第三步不需要再做。
